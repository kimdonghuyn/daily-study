## 📚 모범 답안 & 개념 정리

### 1. 🔧 문제 상황

# 추상화: 인터페이스 vs 추상클래스

## 모범 답안

**이 상황에서는 "추상클래스"를 선택해야 합니다.**

### 선택 이유

1. **공통 속성이 존재한다**
   - 체력, 이름, 경험치 같은 **상태 데이터**를 모든 캐릭터가 공유
   - 인터페이스는 상수만 가능하고 인스턴스 변수를 가질 수 없음
   - 추상클래스만이 이들 속성을 선언하고 초기화할 수 있음

2. **공통 구현 로직이 필요하다**
   - 경험치 증가, 체력 회복 같은 공통 메서드를 구현
   - 모든 자식 클래스가 이 구현을 재사용 가능
   - "IS-A" 관계: 모든 클래스는 "캐릭터이다"

3. **접근 제어가 필요하다**
   - `protected` 키워드로 자식 클래스에만 노출
   - 캡슐화를 통한 상태 관리

### 코드 예시

```java
// ✅ 올바른 선택: 추상클래스
abstract class Character {
    // 공통 속성 (모든 캐릭터가 반드시 가져야 함)
    protected String name;
    protected int health;
    protected int experience;
    
    // 생성자: 공통 초기화 로직
    protected Character(String name, int health) {
        this.name = name;
        this.health = health;
        this.experience = 0;
    }
    
    // 공통 구현 메서드 (모든 자식이 재사용)
    public void gainExperience(int exp) {
        this.experience += exp;
        System.out.println(name + "이 " + exp + "의 경험치를 얻었습니다.");
    }
    
    // 추상 메서드 (자식이 구현해야 함)
    abstract void attack();
    abstract void move();
    abstract void useSkill();
}

// 구체적인 캐릭터 구현
class Warrior extends Character {
    private int armor;
    
    public Warrior(String name, int health, int armor) {
        super(name, health); // 부모 생성자 호출
        this.armor = armor;
    }
    
    @Override
    void attack() {
        System.out.println(name + "이 칼로 공격합니다! 데미지: " + (health / 2));
    }
    
    @Override
    void move() {
        System.out.println(name + "이 무겁지만 든든한 발걸음으로 이동합니다.");
    }
    
    @Override
    void useSkill() {
        System.out.println(name + "의 방어도가 올라갑니다!");
    }
}

class Wizard extends Character {
    private int mana;
    
    public Wizard(String name, int health, int mana) {
        super(name, health);
        this.mana = mana;
    }
    
    @Override
    void attack() {
        System.out.println(name + "이 마법 미사일을 발사합니다!");
    }
    
    @Override
    void move() {
        System.out.println(name + "이 신비한 빛과 함께 이동합니다.");
    }
    
    @Override
    void useSkill() {
        System.out.println(name + "이 마법을 시전합니다! 마나: " + mana);
        mana -= 20;
    }
}

// 사용 예시
public class Game {
    public static void main(String[] args) {
        Character warrior = new Warrior("아서", 100, 30);
        Character wizard = new Wizard("머린", 50, 100);
        
        // 공통 메서드 사용
        warrior.gainExperience(50);  // 모든 캐릭터가 동일
        wizard.gainExperience(30);
        
        // 각자 다르게 구현된 메서드
        warrior.attack();  // "아서이 칼로 공격합니다!"
        wizard.attack();   // "머린이 마법 미사일을 발사합니다!"
    }
}
```

## 핵심 개념 정리

### 📌 추상클래스 (Abstract Class)

| 특징 | 설명 |
|------|------|
| **상태(State)** | ✅ 인스턴스 변수 가능 |
| **생성자** | ✅ 생성자 작성 가능 |
| **접근제어** | ✅ `public`, `protected`, `private` 모두 가능 |
| **메서드 구현** | ✅ 추상 메서드 + 구체적 메서드 혼합 |
| **상속** | `extends` (단일 상속만 가능) |
| **사용 시점** | "같은 종류의 것들" (IS-A 관계) |

### 📌 인터페이스 (Interface)

| 특징 | 설명 |
|------|------|
| **상태(State)** | ❌ 상수만 가능 (불변) |
| **생성자** | ❌ 생성자 불가 |
| **접근제어** | ⚠️ 모두 `public` |
| **메서드 구현** | Java 8+: default 메서드 가능하지만 기본은 추상 메서드 |
| **구현** | `implements` (다중 구현 가능) |
| **사용 시점** | "할 수 있는 것" (CAN-DO 관계) |

// ✅ 기본 캐릭터 계층
abstract class Character {
    protected String name;
    protected int maxHealth;
    protected int currentHealth;
    protected Vector2 position;
    protected int level = 1;
    
    // 공통 로직
    public void takeDamage(int damage) {
        currentHealth = Math.max(0, currentHealth - damage);
    }
    
    public void heal(int amount) {
        currentHealth = Math.min(maxHealth, currentHealth + amount);
    }
    
    abstract void attack();
    abstract void useSkill();
}

// ✅ 역할별 추상클래스 (선택사항: 계층 확장)
abstract class Melee extends Character {
    protected int armor;
    
    @Override
    public void takeDamage(int damage) {
        int actualDamage = Math.max(1, damage - armor / 2);
        super.takeDamage(actualDamage);
    }
}

// ✅ 인터페이스로 능력 추가 (다중 구현)
interface Attackable {
    void attack();
}

interface Healable {
    void heal(int amount);
}

interface Buffable {
    void applyBuff(String buffName);
}

// ✅ 구체적 구현

    @Override
    void attack() {

    }
    
    @Override
    void useSkill() {
        // 방어 자세
    }
    
    @Override
    public void applyBuff(String buffName) {
        // 버프 적용
    }
}
