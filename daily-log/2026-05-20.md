# 캡슐화와 접근 제어자 학습 정리

## ✅ 핵심 개념 요약
캡슐화는 데이터와 메서드를 하나의 단위로 묶고, 접근 제어자를 통해 외부 접근을 제한하는 객체지향 원칙입니다. 이를 통해 데이터 무결성을 보호하고 유지보수성을 높일 수 있습니다.

## 🔑 놓치지 말아야 할 포인트
- **접근 제어자 4가지**: `public` → `protected` → `default` → `private` (공개도 순서)
- **getter/setter 패턴**: private 변수에 접근하기 위한 표준적인 방법, 유효성 검사 추가 가능
- **실무 관점**: 필요한 만큼만 공개하기 (최소 권한 원칙)
