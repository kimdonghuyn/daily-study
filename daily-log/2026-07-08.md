### 1. 🔧 문제 상황

# 상속과 다형성 - 배달 앱 사례 분석

## 모범 답안

이 상황은 **상속(Inheritance)**과 **다형성(Polymorphism)**을 활용하는 전형적인 예시입니다.

### 구조 설계

```python
# 1. 기본 클래스 (부모 클래스)
class Delivery:
    """모든 배달원의 기본 틀"""
    def __init__(self, name: str, location: str):
        self.name = name
        self.location = location
    
    def calculate_delivery_time(self, distance: float) -> float:
        """각 배달원이 구현해야 할 메서드"""
        raise NotImplementedError("서브클래스에서 구현해야 합니다")
    
    def get_delivery_info(self) -> dict:
        """공통 로직"""
        return {
            "name": self.name,
            "location": self.location,
            "delivery_time": self.calculate_delivery_time(10)  # 10km 기준
        }

# 2. 자동차 배달원 (자식 클래스)
class CarDelivery(Delivery):
    """자동차를 이용한 배달"""
    def __init__(self, name: str, location: str, fuel_type: str):
        super().__init__(name, location)
        self.fuel_type = fuel_type
    
    def calculate_delivery_time(self, distance: float) -> float:
        """자동차는 거리 / 평균속도(40km/h) + 신호대기시간"""
        base_time = (distance / 40) * 60  # 분 단위
        wait_time = distance * 0.5  # 신호 대기
        return base_time + wait_time

# 3. 자전거 배달원 (자식 클래스)
class BikeDelivery(Delivery):
    """자전거를 이용한 배달"""
    def __init__(self, name: str, location: str, bike_type: str):
        super().__init__(name, location)
        self.bike_type = bike_type
    
    def calculate_delivery_time(self, distance: float) -> float:
        """자전거는 거리 / 평균속도(15km/h) + 체력 고려"""
        base_time = (distance / 15) * 60  # 분 단위
        fatigue_time = distance * 1.5  # 거리가 길수록 피로도 증가
        return base_time + fatigue_time
```

### 실제 사용 (다형성의 힘)

```python
# 다양한 배달원들을 동일하게 처리
deliveries: list[Delivery] = [
    CarDelivery("김철수", "강남역", "hybrid"),
    BikeDelivery("박영희", "강남역", "road"),
    CarDelivery("이순신", "강남역", "sedan"),
]

def dispatch_deliveries(delivery_list: list[Delivery], distance: float) -> dict:
    """배달원의 타입을 구분하지 않고 처리"""
    results = {}
    
    for delivery in delivery_list:
        # 같은 메서드 호출이지만 실제로는 각각의 구현이 실행됨
        time = delivery.calculate_delivery_time(distance)
        results[delivery.name] = {
            "type": delivery.__class__.__name__,
            "estimated_time": f"{time:.1f}분",
            "info": delivery.get_delivery_info()
        }
    
    return results

# 호출
result = dispatch_deliveries(deliveries, distance=5)
# 출력:
# {
#     "김철수": {"type": "CarDelivery", "estimated_time": "10.5분", ...},
#     "박영희": {"type": "BikeDelivery", "estimated_time": "25.0분", ...},
#     "이순신": {"type": "CarDelivery", "estimated_time": "10.5분", ...}
# }
```

## 핵심 개념 정리

### 📌 상속 (Inheritance)
- **정의**: 기존 클래스의 속성과 메서드를 새 클래스가 물려받는 메커니즘
- **장점**:
  - ✅ 코드 중복 제거 (DRY 원칙)
  - ✅ 공통 로직을 한 곳에서 관리
  - ✅ 유지보수 용이성 증대
- **예시**: `CarDelivery`와 `BikeDelivery`는 `Delivery`의 모든 속성/메서드 상속

### 📌 다형성 (Polymorphism)
- **정의**: "같은 인터페이스, 다른 구현" - 같은 메서드 호출이 객체에 따라 다르게 동작
- **타입**:
  - **메서드 오버라이딩**: 부모 메서드를 자식이 재정의
  - **메서드 오버로딩**: 같은 이름의 메서드를 다른 파라미터로 정의 (Python은 제한적)
- **실제 동작**:
  ```python
  delivery.calculate_delivery_time(10)  # 같은 호출이지만
  # CarDelivery라면 → 자동차 계산식
  # BikeDelivery라면 → 자전거 계산식
  ```

### 📌 관계의 구조
```
        [Delivery] (부모 = 인터페이스)
           /    \
          /      \
   [CarDelivery] [BikeDelivery]  (자식 = 구체적 구현)
```

### 📌 동적 바인딩 (Dynamic Binding)
- 런타임에 실제 객체의 타입에 따라 메서드가 결정됨
- 컴파일 타임에는 `Delivery`인 줄 알지만, 실행 시 `CarDelivery`의 구현 실행

        self.name = name
        self.type = delivery_type
