### 상속과 다형성의 역할

상속과 다형성은 **공통된 특성을 추상화하고, 다양한 구현을 통일된 인터페이스로 다루는** 문제를 해결합니다.

```java
// 1단계: 공통의 추상화 - 상속
public abstract class SchoolMember {
    protected String name;
    protected String id;
    
    public abstract void work();
    public abstract String getRole();
}

public class Teacher extends SchoolMember {
    @Override
    public void work() {
        System.out.println(name + "이(가) 수업을 진행합니다");
    }
    
    @Override
    public String getRole() {
        return "선생님";
    }
}

public class Student extends SchoolMember {
    @Override
    public void work() {
        System.out.println(name + "이(가) 수업을 듣습니다");
    }
    
    @Override
    public String getRole() {
        return "학생";
    }
}

public class Principal extends SchoolMember {
    @Override
    public void work() {
        System.out.println(name + "이(가) 학교를 관리합니다");
    }
    
    @Override
    public String getRole() {
        return "교감선생님";
    }
}

// 2단계: 다형성으로 통일된 처리
public class SchoolSystem {
    private List<SchoolMember> members = new ArrayList<>();
    
    public void addMember(SchoolMember member) {
        members.add(member);
    }
    
    // 모든 구성원을 동일하게 처리
    public void startDay() {
        for (SchoolMember member : members) {
            member.work();  // 다형성: 실제 객체 타입에 따라 다른 동작
            System.out.println("역할: " + member.getRole());
        }
    }
}

// 사용
SchoolSystem school = new SchoolSystem();
school.addMember(new Teacher());
school.addMember(new Student());
school.addMember(new Principal());
school.startDay();  // 각각 다른 work() 구현이 실행됨
```

## 핵심 개념 정리

### 상속 (Inheritance)
- **정의**: 부모 클래스의 특성(메서드, 필드)을 자식 클래스가 물려받는 메커니즘
- **목적**: 공통된 특성을 한 곳에서 관리
- **IS-A 관계**: Student IS-A SchoolMember (학생은 학교 구성원이다)

### 다형성 (Polymorphism)
- **정의**: 같은 메서드 호출이 객체의 실제 타입에 따라 다르게 동작
- **종류**:
  - **메서드 오버라이딩**: 부모 메서드를 자식이 재구현
  - **메서드 오버로딩**: 같은 이름, 다른 파라미터
- **목적**: 유연하고 확장 가능한 코드

### 상속의 핵심 이점
| 문제 | 해결책 |
|------|--------|
| 코드 중복 | 공통 로직을 부모 클래스에 작성 |
| 유지보수 어려움 | 부모 클래스만 수정하면 모든 자식에 반영 |
| 새로운 타입 추가 시 기존 코드 수정 필요 | 새 자식 클래스만 추가하면 됨 |
| 다양한 타입을 개별적으로 처리 | 부모 타입으로 통일하여 처리 |

### 다형성의 핵심 이점
```
"동일한 인터페이스로 다양한 객체를 다룬다"
= 코드가 확장에 열려있고 수정에 닫혀있다 (OCP 원칙)
```
