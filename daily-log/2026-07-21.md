# 상속과 다형성을 활용한 배달 앱 설계

## 모범 답안

### 핵심 전략: "인터페이스 기반 설계"로 확장성 확보

```java
// 1단계: 모든 음식이 따를 공통 계약(인터페이스)
public interface Food {
    void prepare();           // 준비 과정
    int getPreparationTime(); // 준비 시간
    double getPrice();        // 가격
}

// 2단계: 각 음식별 구체적 구현 (다형성)
public class Pizza implements Food {
    private int cheeseAmount;
    
    public Pizza(int cheeseAmount) {
        this.cheeseAmount = cheeseAmount;
    }
    
    @Override
    public void prepare() {
        System.out.println("피자 준비: 치즈 " + cheeseAmount + "배");
        System.out.println("170도에서 12분 구우기");
    }
    
    @Override
    public int getPreparationTime() {
        return 12;
    }
    
    @Override
    public double getPrice() {
        return 12000 + (cheeseAmount * 1000);
    }
}

public class Noodles implements Food {
    private String noodleThickness;
    
    public Noodles(String thickness) {
        this.noodleThickness = thickness;
    }
    
    @Override
    public void prepare() {
        System.out.println("국수 준비: 면의 굵기 " + noodleThickness);
        System.out.println("끓는 물에 3분 삶기");
    }
    
    @Override
    public int getPreparationTime() {
        return 3;
    }
    
    @Override
    public double getPrice() {
        return 8000;
    }
}

public class Burger implements Food {
    private int pattyCount;
    
    public Burger(int patties) {
        this.pattyCount = patties;
    }
    
    @Override
    public void prepare() {
        System.out.println("버거 준비: 패티 " + pattyCount + "개");
        System.out.println("각 패티 5분씩 구우면서 조립");
    }
    
    @Override
    public int getPreparationTime() {
        return 5 * pattyCount;
    }
    
    @Override
    public double getPrice() {
        return 8000 + (pattyCount * 2000);
    }
}

// 3단계: 배달 시스템 (Food 인터페이스만 의존)
public class DeliverySystem {
    private List<Food> orders = new ArrayList<>();
    
    // ✅ 새로운 음식이 추가되어도 이 코드는 수정 불필요!
    public void addOrder(Food food) {
        orders.add(food);
    }
    
    public void prepareAllOrders() {
        for (Food food : orders) {
            food.prepare();  // 각 음식의 고유한 prepare() 호출
        }
    }
    
    public int getTotalPreparationTime() {
        return orders.stream()
            .mapToInt(Food::getPreparationTime)
            .sum();
    }
    
    public double calculateTotalPrice() {
        return orders.stream()
            .mapToDouble(Food::getPrice)
            .sum();
    }
}

// 4단계: 실제 사용
public class Main {
    public static void main(String[] args) {
        DeliverySystem delivery = new DeliverySystem();
        
        delivery.addOrder(new Pizza(2));           // 치즈 2배 피자
        delivery.addOrder(new Noodles("굵음"));    // 굵은 면
        delivery.addOrder(new Burger(3));          // 3패티 버거
        
        delivery.prepareAllOrders();
        System.out.println("총 준비시간: " + delivery.getTotalPreparationTime() + "분");
        System.out.println("총 가격: " + delivery.calculateTotalPrice() + "원");
    }
}
```

**출력:**
```

170도에서 12분 구우기
국수 준비: 면의 굵기 굵음
끓는 물에 3분 삶기
버거 준비: 패티 3개
각 패티 5분씩 구우면서 조립
총 준비시간: 20분
총 가격: 37000원
```

### 핵심 메커니즘

| 요소 | 역할 |
|------|------|
| **Food 인터페이스** | 모든 음식이 반드시 구현해야 할 "계약" |
| **각 음식 클래스** | 자신의 방식대로 prepare() "구현" (오버라이딩) |
| **DeliverySystem** | Food 타입만 알고, 구체적 음식은 신경 쓰지 않음 |
| **런타임 다형성** | 실행 시점에 실제 객체의 메서드가 호출됨 |

## 핵심 개념 정리

### 🎯 상속 vs 인터페이스
- **상속**: "is-a" 관계 (Pizza **is a** Food)
- **인터페이스**: "can-do" 계약 (모든 Food는 prepare할 수 있다)
- **권장**: 배달앱처럼 다양한 구현이 필요하면 **인터페이스** 사용

### 🔄 다형성의 3가지 요소

```java
// 1. 인터페이스 정의 (추상화)
Food food = ...;

// 2. 다양한 구현체 (상속/구현)
food = new Pizza(2);      // 런타임에 Pizza로 변함
food = new Noodles("가는"); // 런타임에 Noodles로 변함

// 3. 같은 메서드 호출, 다른 동작 (다형성)
food.prepare();  // Pizza면 피자 준비, Noodles면 국수 준비
```

### ⚙️ Open/Closed 원칙 (SOLID)
- **Open for extension**: 새로운 음식 추가 가능 ✅
- **Closed for modification**: DeliverySystem 수정 불필요 ✅

```java
// ❌ 새 음식 추가할 때마다 if-else 추가 (나쁜 예)
if (foodType.equals("pizza")) { ... }
else if (foodType.equals("noodles")) { ... }
else if (foodType.equals("burger")) { ... }
// → 100가지면 코드가 터짐

// ✅ 인터페이스 구현만 추가 (좋은 예)
public class Sushi implements Food { ... }
// → DeliverySystem은 그대로!
```

### 📌 런타임 다형성 메커니즘

```java
Food food = new Pizza(2);  // 컴파일 타입: Food
                            // 런타임 타입: Pizza

food.prepare();  // JVM이 "이건 Pizza다"를 알고
                 // Pizza의 prepare() 호출 (동적 디스패치)
```
