# 캡슐화와 접근 제어자 - 모범 답안

## 모범 답안

### 1단계: 문제 상황 분석

"만약 잔액 정보를 누구나 직접 수정할 수 있다면 심각한 문제가 발생합니다:"

```
❌ 잘못된 예시
account.balance = -1000000;  // 음수 잔액?
account.balance = 999999999; // 임의로 증액?
```

**발생 가능한 문제들:**
- **데이터 무결성 침해**: 부정한 출금, 송금 가능
- **보안 위협**: 권한 없는 수정
- **비즈니스 로직 우회**: 수수료 미징수, 한도 초과 등
- **감시 추적 불가**: 누가, 언제 변경했는지 기록 불가

### 2단계: 접근 제어자의 필요성

"**캡슐화(Encapsulation)**를 통해 데이터를 보호합니다:"

```java
public class BankAccount {
    // ✅ 직접 접근 금지
    private double balance;
    
    // 잔액 조회는 가능하게
    public double getBalance() {
        return balance;
    }
}
```

**접근 제어자의 역할:**
- `private`: 클래스 내부에서만 접근 가능
- `public`: 외부에서 접근 가능
- `protected`, `package-private`: 선택적 공개

### 3단계: 기능 제공 - 공개 인터페이스

"정제된 공개 메서드를 통해서만 접근을 허용합니다:"

```java
public class BankAccount {
    private double balance;
    private List<Transaction> transactionHistory = new ArrayList<>();
    
    // ✅ 읽기는 외부에 공개
    public double getBalance() {
        return balance;
    }
    
    // ✅ 쓰기는 비즈니스 로직을 거쳐서만 가능
    public void deposit(double amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("입금액은 양수여야 합니다");
        }
        balance += amount;
        recordTransaction("입금", amount);
    }
    
    public boolean withdraw(double amount) {
        if (amount > balance) {
            return false;  // 잔액 부족
        }
        if (amount <= 0) {
            throw new IllegalArgumentException("출금액은 양수여야 합니다");
        }
        balance -= amount;
        recordTransaction("출금", amount);
        return true;
    }
    
    // ✅ 변경 내역 추적
    private void recordTransaction(String type, double amount) {
        transactionHistory.add(
            new Transaction(type, amount, LocalDateTime.now())
        );
    }
}
```

**사용 예시:**
```java
BankAccount account = new BankAccount();

// ✅ 허용된 방식
account.deposit(10000);
account.withdraw(5000);
System.out.println(account.getBalance());  // 5000

// ❌ 불가능한 방식
// account.balance = -1000000;  // 컴파일 에러!
```

## 핵심 개념 정리

### 🔐 접근 제어자 (Access Modifiers)

| 제어자 | 클래스 | 패키지 | 자식클래스 | 외부 |
|--------|--------|--------|-----------|------|
| `private` | ✅ | ❌ | ❌ | ❌ |
| `package-private` (기본값) | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ |

### 📦 캡슐화 (Encapsulation)

- **데이터 은닉 (Data Hiding)**: 내부 상태를 `private`으로 숨김
- **공개 인터페이스**: 필요한 기능만 `public` 메서드로 노출
- **제어된 접근**: 모든 수정이 비즈니스 로직을 통함

### ✨ 관련 원칙들

| 개념 | 설명 |
|------|------|
| **정보 은닉** | 필요한 것만 외부에 공개 |
| **단일 책임** | 각 메서드는 하나의 책임만 |
| **불변식 보호** | 항상 유효한 상태 유지 |
| **계약 기반 설계** | 메서드의 전제조건, 후제조건 명확 |

### 1️⃣ Getter/Setter 패턴의 올바른 사용

```java
// ❌ 안티패턴: 단순 get/set은 캡슐화 의미 없음
public double getBalance() { return balance; }
public void setBalance(double balance) { 
    this.balance = balance;  // 누구나 수정 가능!
}

// ✅ 올바른 방식: 비즈니스 로직 포함
public double getBalance() { return balance; }
// setBalance는 없음. withdraw/deposit으로만 변경 가능
```

### 2️⃣ 불변성 고려

```java
public class BankAccount {
    private final List<Transaction> history = new ArrayList<>();
    
    // ❌ 위험: 내부 컬렉션 직접 노출
    public List<Transaction> getHistory() {
        return history;  // 호출자가 수정 가능!
    }
    
    // ✅ 안전: 불변 복사본 반환
    public List<Transaction> getHistory() {
        return Collections.unmodifiableList(history);
    }
    
    // 또는 복사본 반환
    public List<Transaction> getHistory() {
        return new ArrayList<>(history);
    }
}
```

### 4️⃣ 다층 방어 (Defense in Depth)

```java
public class BankAccount {
    private double balance;
    
    public void withdraw(double amount) {
        // 1단계: 입력 검증
        if (amount <= 0) {
            throw new IllegalArgumentException("양수만 가능");
        }
        
        // 2단계: 비즈니스 규칙 검증
        if (amount > balance) {
            throw new InsufficientBalanceException();
        }
        
        // 3단계: 권한 검증
        if (!currentUser.hasPermission(this)) {
            throw new AccessDeniedException();
        }
        
        // 4단계: 실행
        balance -= amount;
        
        // 5단계: 감시 및 기록
        auditLog.record(currentUser, "출금", amount);
    }
}
```

public class MoneyTransfer {
