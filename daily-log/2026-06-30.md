# 캡슐화와 접근 제어자 학습 정리

## ✅ 핵심 개념 요약
**캡슐화**는 객체의 내부 상태를 숨기고 필요한 인터페이스만 공개하는 원칙입니다. **접근 제어자**(public, private, protected, default)를 통해 클래스 멤버의 접근 범위를 제한하여 데이터 무결성과 보안을 보장합니다.

## 🔑 놓치지 말아야 할 포인트
- **private**: 같은 클래스 내에서만 접근 가능 → getter/setter로 제어된 접근 제공
- **protected vs default**: 상속 관계에서 접근 범위 차이 (protected는 패키지 외 상속 클래스에서도 접근 가능)
- **캡슐화의 이점**: 변수 이름 변경 시 내부만 수정 → 외부 코드 영향 최소화
