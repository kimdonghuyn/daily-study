# 캡슐화와 접근 제어자 학습 정리

## ✅ 핵심 개념 요약
**캡슐화**는 객체의 내부 상태를 숨기고 필요한 인터페이스만 노출하는 원칙입니다. **접근 제어자**(public, protected, private 등)를 통해 캡슐화를 구현하여 데이터 무결성을 보장하고 유지보수성을 높입니다.

## 🔑 놓치지 말아야 할 포인트
- **접근 제어자별 범위**: private(자신만) < protected(상속) < public(모두) 이해
- **getter/setter의 필요성**: 단순 접근이 아닌 검증 로직을 통한 데이터 보호
- **정보 은닉의 이점**: 내부 구현 변경 시 외부 영향 최소화 → 유지보수성 증대
