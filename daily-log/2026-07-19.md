### 1. 🔧 문제 상황

# 상속과 다형성 - 모범 답안 및 개념 정리

## 모범 답안

이 상황은 **상속과 다형성**이 정확히 해결해야 할 문제입니다.

**문제 분석:**
- 플레이어, 몬스터, 보스는 모두 "캐릭터"라는 공통의 개념
- 공통 기능(체력 깎기, 스킬 사용, 죽기)이 반복되고 있음
- 규모 확장 시 유지보수 복잡도 증가

**해결 방안:**

```python
# 1️⃣ 상속을 통한 공통 코드 통합
class Character:
    """모든 캐릭터의 기본 동작을 정의"""
    def __init__(self, name: str, hp: int):
        self.name = name
        self.hp = hp
        self.max_hp = hp
    
    def take_damage(self, damage: int) -> None:
        """모든 캐릭터가 동일하게 체력 깎기"""
        self.hp = max(0, self.hp - damage)
        print(f"{self.name}이(가) {damage}의 피해를 입었습니다. (현재 체력: {self.hp})")
    
    def use_skill(self) -> None:
        """다형성을 위한 기본 메서드 (오버라이드 예정)"""
        raise NotImplementedError("use_skill은 반드시 구현되어야 합니다")
    
    def die(self) -> None:
        """사망 처리"""
        if self.hp <= 0:
            print(f"{self.name}이(가) 죽었습니다.")
            return True
        return False

# 2️⃣ 다형성을 통한 각 캐릭터별 맞춤 동작
class Player(Character):
    """플레이어 전용 기능"""
    def use_skill(self) -> None:
        print(f"{self.name}이(가) [파이어볼] 스킬을 사용했습니다!")
        return 50  # 데미지 반환

class Monster(Character):
    """일반 몬스터"""
    def use_skill(self) -> None:
        print(f"{self.name}이(가) [스크래치] 공격을 했습니다!")
        return 15

class Boss(Character):
    """보스 몬스터 - 특별한 동작"""
    def __init__(self, name: str, hp: int, phase: int = 1):
        super().__init__(name, hp)
        self.phase = phase
    
    def use_skill(self) -> None:
        if self.phase == 1:
            print(f"{self.name}이(가) [울부짖음]을 사용했습니다!")
            return 30
        else:
            print(f"{self.name}이(가) [궁극 기술]을 사용했습니다!")
            return 80

# 3️⃣ 실제 사용 (다형성의 강력함)
def battle_round(attacker: Character, defender: Character) -> None:
    """어떤 캐릭터든 동일한 방식으로 처리"""
    damage = attacker.use_skill()
    defender.take_damage(damage)
    if defender.die():
        pass  # 전투 종료

# 사용 예시
player = Player("영웅", hp=100)
goblin = Monster("고블린", hp=30)
dragon = Boss("드래곤", hp=200)

# 모두 동일한 인터페이스로 처리 가능
characters: list[Character] = [player, goblin, dragon]

for character in characters:
    character.use_skill()  # 각각 다른 동작 수행!
```

**핵심 이점:**
✅ **코드 중복 제거** - `take_damage`, `die` 한 번만 작성  
✅ **확장성** - 새 캐릭터 추가 시 `Character` 상속만으로 완료  
✅ **유지보수** - "체력 깎기" 로직 변경 시 `Character` 클래스만 수정  
✅ **유연성** - 각 캐릭터는 고유한 `use_skill()` 구현 가능

## 핵심 개념 정리

### 🏛️ **상속 (Inheritance)**
- **정의**: 부모 클래스의 속성과 메서드를 자식 클래스가 물려받음
- **목적**: 공통 코드 통합, DRY(Don't Repeat Yourself) 원칙
- **문법**: `class Child(Parent):`
- **특징**:
  - 단일 상속 vs 다중 상속
  - `super()` 를 통한 부모 메서드 호출
  - 생성자 오버라이딩

### 🎭 **다형성 (Polymorphism)**
- **정의**: "같은 이름, 다른 동작" - 하나의 인터페이스로 여러 구현 수행
- **메커니즘**:
  1. **메서드 오버라이딩** (가장 중요)
     ```python
     class Parent:
         def action(self): pass
     
     class Child1(Parent):
         def action(self): print("Child1의 동작")
     
     class Child2(Parent):
         def action(self): print("Child2의 동작")
     ```
  
  2. **메서드 오버로딩** (파이썬에선 제한적)
     ```python
     # 파이썬은 기본적으로 오버로딩 미지원
     # 가변 인자로 구현
     def action(self, *args):
         pass
     ```

### 🔄 **LSP (Liskov Substitution Principle)**
- 부모 타입이 필요한 곳에 자식 타입을 대체 가능해야 함
- `character: Character = Player(...)` ✅ 가능
- 동일한 인터페이스로 모든 자식 객체 처리
