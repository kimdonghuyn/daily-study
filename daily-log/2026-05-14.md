# JVM 메모리 구조 학습 정리

✅ **핵심 개념 요약**
JVM은 프로그램 실행 시 메모리를 Stack(지역변수, 메서드 호출), Heap(객체 인스턴스), Method Area(클래스 정보, 메서드 코드)로 분리 관리합니다. 각 영역은 생명주기와 용도가 다르며, 메모리 누수와 성능 최적화를 위해 이해가 필수적입니다.

🔑 **놓치지 말아야 할 포인트**
- **Stack**: 스레드별 독립 할당, LIFO 방식, 메서드 종료 시 자동 해제
- **Heap**: 모든 스레드 공유, Garbage Collection 대상, 메모리 누수의 주요 원인
- **Method Area**: 클래스 로딩 시 한 번만 할당, 메서드/상수 데이터 저장

📚 **내일 복습 포인트**
- Garbage Collection 동작 원리 (GC 알고리즘)
- Stack Overflow vs Heap OutOfMemory 차이와 발생 원인
- 메모리 누수 케이스 실습 (예: 순환 참조, 정적 변수 오용)
