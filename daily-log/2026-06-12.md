## 📚 모범 답안 & 개념 정리

### 1. 📖 개념 설명

# Spring @Transactional의 내부 동작 원리

## 모범 답안

Spring의 @Transactional은 **프록시 패턴과 AOP(Aspect Oriented Programming)**를 결합하여 구현됩니다.

### 1단계: 프록시 객체 생성

```
원본 Bean (UserService)
         ↓
Spring Container가 감지 (@Transactional 어노테이션)
         ↓
프록시 객체 생성 (TransactionInterceptor 포함)
         ↓
컨테이너에 프록시 객체 등록 (원본은 등록 안됨)
```

### 2단계: 메서드 호출 시 실제 흐름

```java
// 우리가 작성한 코드
@Service
public class UserService {
    @Transactional
    public void transferMoney(Long fromId, Long toId, BigDecimal amount) {
        // 비즈니스 로직만 작성
        userRepository.updateBalance(fromId, -amount);
        userRepository.updateBalance(toId, amount);
    }
}

// Spring이 실제로 생성하는 프록시 객체의 의사 코드
public class UserServiceProxy extends UserService {
    private TransactionManager transactionManager;
    
    @Override
    public void transferMoney(Long fromId, Long toId, BigDecimal amount) {
        // 1. 트랜잭션 시작
        Transaction tx = transactionManager.begin();
        try {
            // 2. 원본 메서드 실행
            super.transferMoney(fromId, toId, amount);
            
            // 3. 성공 시 커밋
            transactionManager.commit(tx);
        } catch (Exception e) {
            // 4. 예외 발생 시 롤백
            transactionManager.rollback(tx);
            throw e;
        }
    }
}
```

### 3단계: 의존성 주입 시

```java
// 실제로는 프록시 객체가 주입됨
@Autowired
private UserService userService; // ← 실제로는 Proxy 객체

// 호출 시
userService.transferMoney(...); // ← Proxy의 메서드 호출
                                 // ← 내부에서 원본 메서드 호출
```

## 핵심 개념 정리

### 📌 프록시 패턴 (Proxy Pattern)
- **목적**: 원본 객체에 접근하기 전에 **중간 객체를 거쳐 접근 제어**
- **장점**: 원본 코드 수정 없이 기능 추가 가능
- **구현 방식**: 
  - JDK Dynamic Proxy (인터페이스 기반)
  - CGLIB Proxy (상속 기반)

### 📌 AOP (Aspect Oriented Programming)
```
Cross-Cutting Concern을 분리하는 기법

┌─────────────────────────────────────┐
│         핵심 로직 (Business Logic)    │ ← @Transactional이 없었다면
│  • 돈 이체                            │   모든 메서드에 직접 작성해야 함
│  • 유저 생성                          │
│  • 주문 처리                          │
└─────────────────────────────────────┘
           ↓↑ (위아래로 감싸짐)
┌─────────────────────────────────────┐
│    횡단 관심사 (Cross-Cutting)       │ ← AOP로 분리
│  • 트랜잭션 관리                      │
│  • 보안 검증                         │
│  • 로깅 / 모니터링                   │
│  • 성능 측정                         │
└─────────────────────────────────────┘
```

### 📌 Spring AOP 용어
| 용어 | 설명 |
|------|------|
| **Aspect** | 횡단 관심사를 모듈화한 것 (트랜잭션 관리 = Aspect) |
| **JoinPoint** | AOP가 적용될 수 있는 포인트 (메서드 호출 시점) |
| **Pointcut** | 실제로 AOP를 적용할 대상 (@Transactional 메서드들) |
| **Advice** | 적용할 로직 (Before, After, Around 등) |
| **Weaving** | Aspect를 타겟 객체에 적용하는 과정 |

### 📌 Spring이 프록시를 선택하는 기준

```java
// Case 1: 인터페이스 구현 → JDK Proxy 사용
public interface UserService {
    void transfer(Long from, Long to, BigDecimal amount);
}
public class UserServiceImpl implements UserService { ... }
// ↓ JDK Proxy 생성 (인터페이스 기반)

// Case 2: 인터페이스 미구현 → CGLIB Proxy 사용
public class UserService {
    @Transactional
    public void transfer(Long from, Long to, BigDecimal amount) { ... }
}
// ↓ CGLIB Proxy 생성 (상속 기반)
```

### 📌 @Transactional 처리 흐름

```
메서드 호출
    ↓
TransactionInterceptor (Advice)
    ↓
┌─────────────────────────────────────┐
│ 1. TransactionManager.getTransaction()│
│    ↓                                  │
│ 2. 원본 메서드 실행 (proceed())        │
│    ↓                                  │
│ 3. 성공 → commit()                    │
│    실패 → rollback()                  │
└─────────────────────────────────────┘
    ↓
반환
```

## 실무 포인트

### ⚡ 1. 프록시 패턴 주의사항

```java
// ❌ 위험: 같은 클래스 내에서 호출하면 프록시를 거치지 않음
@Service
public class UserService {
    @Transactional
    public void transferMoney(Long from, Long to, BigDecimal amount) {
        updateBalance(from, -amount);  // ← 자체 호출
        updateBalance(to, amount);
    }
    
    @Transactional
    public void updateBalance(Long userId, BigDecimal amount) {
        // 이 트랜잭션은 작동하지 않음!
        // 왜? transferMoney() 내부에서 직접 호출되므로
        // 프록시를 거치지 않기 때문
    }
}

// ✅ 올바른 해결책
@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;
    
    @Transactional
    public void transferMoney(Long from, Long to, BigDecimal amount) {
        // 별도의 메서드 호출 제거
        // repository를 직접 호출
        userRepository.updateBalance(from, -amount);
        userRepository.updateBalance(to, amount);
    }
}

// 또는 ApplicationContext를 주입받아 강제로 프록시 호출
@Service
public class UserService {
    @Autowired
    private ApplicationContext context;
    
    public void transferMoney(Long from, Long to, BigDecimal amount) {
        UserService proxy = context.getBean(UserService.class);
        proxy.updateBalance(from, -amount);  // ← 프록시 호출
        proxy.updateBalance(to, amount);
    }
}
```
