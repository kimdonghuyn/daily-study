# 프록시 패턴과 AOP 내부 동작 원리

## 모범 답안

### 현재 상황의 문제점

지금 당신이 하는 방식은 **횡단 관심사(Cross-Cutting Concern)를 비즈니스 로직에 산재**시키는 것입니다. 
- 10개 함수에 로깅 코드 반복 → 코드 중복
- 100개로 늘어나면 → 유지보수 악몽 (로깅 로직 변경 시 100곳 수정)
- 돈 이체 로직 자체가 로깅 코드로 오염됨 → 가독성 및 테스트 어려움

### 프록시 패턴 vs AOP: 무엇이 다른가?

```java
// ❌ 현재 방식 (로깅 코드 산재)
public void transfer(String from, String to, int amount) {
    System.out.println("이체 시작: " + from); // ← 관심사 섞임
    
    // 실제 비즈니스 로직
    validateAccount(from, to);
    deductMoney(from, amount);
    addMoney(to, amount);
    
    System.out.println("이체 완료"); // ← 관심사 섞임
}
```

#### 1️⃣ **프록시 패턴 (Proxy Pattern)** - 구조적 해결
```java
// 원본 객체
public class RealTransferService implements TransferService {
    @Override
    public void transfer(String from, String to, int amount) {
        // 순수 비즈니스 로직만
        validateAccount(from, to);
        deductMoney(from, amount);
        addMoney(to, amount);
    }
}

// 프록시 객체 (책임 분리)
public class TransferServiceProxy implements TransferService {
    private RealTransferService realService;
    
    @Override
    public void transfer(String from, String to, int amount) {
        System.out.println("이체 시작: " + from);
        try {
            realService.transfer(from, to, amount);
            System.out.println("이체 완료");
        } catch (Exception e) {
            System.out.println("이체 실패: " + e.getMessage());
        }
    }
}

// 사용
TransferService service = new TransferServiceProxy(new RealTransferService());
service.transfer("계좌A", "계좌B", 10000);
```

**특징:**
- ✅ 명시적: 어떤 대상에 프록시를 적용했는지 코드에 보임
- ✅ 세밀한 제어: 특정 메서드만 선택적으로 프록시 적용 가능
- ❌ 단점: 프록시 클래스를 명시적으로 작성해야 함
- ❌ 각 서비스마다 프록시 클래스 필요

#### 2️⃣ **AOP (Aspect-Oriented Programming)** - 동적 해결
```java
// 원본 객체 (순수함)
@Service
public class TransferService {
    public void transfer(String from, String to, int amount) {
        validateAccount(from, to);
        deductMoney(from, amount);
        addMoney(to, amount);
    }
}

// Aspect 정의 (횡단 관심사)
@Aspect
@Component
public class TransferLoggingAspect {
    
    @Before("@annotation(Loggable)") // 실행 전
    public void beforeTransfer(JoinPoint joinPoint) {
        System.out.println("이체 시작: " + joinPoint.getArgs()[0]);
    }
    
    @After("@annotation(Loggable)") // 실행 후
    public void afterTransfer(JoinPoint joinPoint) {
        System.out.println("이체 완료");
    }
    
    @AfterThrowing("@annotation(Loggable)") // 예외 발생 시
    public void handleException(JoinPoint joinPoint, Exception e) {
        System.out.println("이체 실패: " + e.getMessage());
    }
}

// 적용
@Service
public class TransferService {
    
    @Loggable  // ← annotation만 붙이면 됨
    public void transfer(String from, String to, int amount) {
        validateAccount(from, to);
        deductMoney(from, amount);
        addMoney(to, amount);
    }
    
    @Loggable
    public void transferInternational(String from, String to, int amount) {
        // ...
    }
    // 100개까지 늘어도 그냥 @Loggable만 붙이면 됨!
}
```

**특징:**
- ✅ 선언적: annotation 하나로 모든 메서드에 적용
- ✅ 확장성: 함수 개수 증가해도 코드 변경 없음
- ✅ 중앙집중식: 모든 로깅 로직이 Aspect에 모여 있음
- ⚠️ 약간의 마법: Spring이 런타임에 프록시를 자동 생성
- ⚠️ 디버깅 복잡도: 내부 동작을 모르면 헷갈림

## 핵심 개념 정리

### 🎯 **프록시 패턴 (Proxy Pattern)**
- **정의**: 다른 객체를 대리(대신)하는 객체를 생성해 요청을 중간에서 처리
- **생성 시점**: 개발 시 (명시적으로 코드에 작성)
- **적용 범위**: 특정 클래스/메서드 명확
- **사용 사례**: DB connection pooling, lazy loading, access control
- **GoF 디자인 패턴**: 구조적 패턴(Structural Pattern)

```java
// Proxy 역할
class DatabaseConnectionProxy implements Connection {
    private Connection realConnection; // 실제 객체
    
    public void connect() {
        System.out.println("연결 풀에서 가져옴");
        realConnection.connect();
    }
}
```

### 🎯 **AOP (Aspect-Oriented Programming)**
- **정의**: 관심사의 분리(SoC)를 달성하기 위해 횡단 관심사를 모듈화
- **생성 시점**: 런타임 (Spring Container가 자동 생성)
- **적용 범위**: pointcut으로 동적으로 결정
- **사용 사례**: 로깅, 트랜잭션, 보안, 캐싱, 성능 모니터링
- **핵심 개념**:
  - **Aspect**: 횡단 관심사 모듈 (로깅, 트랜잭션 등)
  - **Joinpoint**: Aspect가 적용될 수 있는 모든 지점
  - **Pointcut**: Joinpoint 중 실제 적용할 지점 선택
  - **Advice**: Pointcut에서 실행될 코드 (@Before, @After 등)
  - **Weaving**: Aspect를 실제 코드에 적용하는 과정

```
┌─────────────────────────────────┐
│  Aspect (횡단 관심사)            │
│  ├─ @Before (Advice)            │
│  ├─ @After (Advice)             │
│  └─ Pointcut (적용 대상 선택)    │
└─────────────────────────────────┘
           │
        Weaving
           │
    런타임 프록시 생성
```
