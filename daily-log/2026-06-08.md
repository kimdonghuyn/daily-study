## 📚 모범 답안 & 개념 정리

### 1. 🔧 문제 상황

# Spring Bean 생명주기와 스코프 - 완벽 가이드

## 모범 답안

### 문제의 근본 원인

**Bean 스코프란 Spring Container가 Bean 인스턴스를 생성, 관리, 소멸시키는 범위**입니다.

당신의 경우, UserService가 **기본값인 Singleton 스코프**로 관리되었기 때문에 문제가 발생했습니다:

```
┌─────────────────────────────────────┐
│     Spring Container (Singleton)     │
│                                      │
│  UserService (단 1개 인스턴스)       │
│  ├─ currentUser = "User A"           │
│  │                                   │
│  └─ 모든 요청이 같은 객체 사용       │
│                                      │
│  User A 접속 → currentUser = "A"    │
│  User B 접속 → currentUser = "B"    │ (A의 데이터 덮어씌움)
└─────────────────────────────────────┘
```

### 직관적 설명

**비유로 생각해보면:**
- **Singleton**: 은행에 ATM이 1대만 있는데, 모든 고객이 그 ATM을 공유
  - User A가 100만원 입금 → User B가 꺼내가면 A의 데이터가 사라짐
  
- **Request 스코프**: 각 고객이 자기 전용 ATM을 가짐
  - User A의 ATM, User B의 ATM 별도 존재

### 해결 방법

```java
// ❌ 문제가 되는 코드 (Singleton)
@Service
public class UserService {
    private User currentUser;  // 모든 요청이 공유
    
    public void setCurrentUser(User user) {
        this.currentUser = user;
    }
    
    public User getCurrentUser() {
        return this.currentUser;  // 다른 사용자 데이터 반환 가능
    }
}

// ✅ 해결 방법 1: Request 스코프 사용
@Service
@Scope("request")
public class UserService {
    private User currentUser;  // 각 HTTP 요청마다 새로운 인스턴스
    
    public void setCurrentUser(User user) {
        this.currentUser = user;
    }
    
    public User getCurrentUser() {
        return this.currentUser;  // 안전함
    }
}

// ✅ 해결 방법 2: ThreadLocal 사용 (더 나은 방법)
@Service
public class UserService {
    private final ThreadLocal<User> currentUser = new ThreadLocal<>();
    
    public void setCurrentUser(User user) {
        currentUser.set(user);
    }
    
    public User getCurrentUser() {
        return currentUser.get();
    }
    
    public void clear() {
        currentUser.remove();  // 메모리 누수 방지
    }
}

// ✅ 해결 방법 3: SecurityContext 사용 (가장 권장)
@Service
public class UserService {
    public User getCurrentUser() {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        return (User) auth.getPrincipal();  // 스레드별로 자동 관리
    }
}
```

## 핵심 개념 정리

### 1️⃣ **Spring Bean 스코프의 종류**

| 스코프 | 생명주기 | 사용처 | 특징 |
|--------|--------|--------|------|
| **Singleton** | 애플리케이션 시작~종료 | 일반 서비스, DAO | 1개 인스턴스만 생성 (기본값) |
| **Prototype** | 요청할 때마다 | 상태를 가진 객체 | 매번 새로운 인스턴스 생성 |
| **Request** | HTTP 요청 시작~종료 | 웹 요청별 데이터 | 각 요청마다 새 인스턴스 |
| **Session** | HTTP 세션 시작~종료 | 사용자 세션 데이터 | 세션당 1개 인스턴스 |
| **Application** | ServletContext 시작~종료 | 애플리케이션 전역 데이터 | 전체 애플리케이션 공유 |
| **WebSocket** | WebSocket 세션 | 실시간 통신 | 웹소켓 연결당 1개 |

### 2️⃣ **Singleton의 문제점**

```java
@Service  // 기본값: @Scope("singleton")
public class UserService {
    private User currentUser;  // 공유 상태(Shared State)
    
    // 문제: 멀티스레드 환경에서 Race Condition 발생
    // Thread 1: currentUser = User("A")
    // Thread 2: currentUser = User("B")  <- A의 데이터 손실
}
```

**Race Condition 발생 과정:**
```
Time  Thread1              Thread2
t1    읽기: user = null    -
t2    -                    읽기: user = null
t3    쓰기: user = "A"     -
t4    -                    쓰기: user = "B" (A 덮어씌움)
```

### 3️⃣ **Bean 생명주기 (Lifecycle)**

```
┌─────────────────────────────────────────┐
│  1. Bean 인스턴스 생성 (Constructor)    │
├─────────────────────────────────────────┤
│  2. 의존성 주입 (Dependency Injection)  │
├─────────────────────────────────────────┤
│  3. @PostConstruct 메서드 호출          │
├─────────────────────────────────────────┤
│  4. Bean 사용 (비즈니스 로직)           │
├─────────────────────────────────────────┤
│  5. @PreDestroy 메서드 호출             │
├─────────────────────────────────────────┤
│  6. Bean 인스턴스 소멸                  │
└─────────────────────────────────────────┘
```

```java
@Service
public class UserService implements InitializingBean, DisposableBean {
    private DataSource dataSource;
    
    // 방법 1: Lifecycle Interface 구현
    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("Bean 초기화됨");
        // 리소스 초기화
    }
    
    @Override
    public void destroy() throws Exception {
        System.out.println("Bean 소멸됨");
        // 리소스 정리
    }
    
    // 방법 2: 어노테이션 (권장)
    @PostConstruct
    public void init() {
        System.out.println("초기화 로직");
    }
    
    @PreDestroy
    public void cleanup() {
        System.out.println("정리 로직");
    }
}
```

### 4️⃣ **Request & Session 스코프의 동작**

```java
@Service
@Scope(value = "request", proxyMode = ScopedProxyMode.TARGET_CLASS)
public class UserService {
    // 각 HTTP 요청마다 새로운 인스턴스가 생성됨
    private User currentUser;
}

// 또는 WebRequest 사용
@Service
public class UserService {
    @Autowired
    private WebRequest request;
    
    public void setCurrentUser(User user) {
        request.setAttribute("currentUser", user, WebRequest.SCOPE_REQUEST);
    }
    
    public User getCurrentUser() {
        return (User) request.getAttribute("currentUser", WebRequest.SCOPE_REQUEST);
    }
}
```
