### 1. 🔧 문제 상황

# JVM 메모리 구조 - Heap vs Stack OutOfMemoryError

## 모범 답안

**Java Heap space 에러는 객체 생성 과다로 발생하고, Stack 에러는 호출 깊이 과다로 발생합니다.**

### Heap이 터지는 경우 (Java Heap space)
```java
// ❌ 대량 객체 생성 - Heap 고갈
List<byte[]> list = new ArrayList<>();
while (true) {
    list.add(new byte[1024 * 1024]); // 1MB씩 계속 생성
    // GC가 이들을 회수할 수 없음 (list가 계속 참조)
}

// ❌ 메모리 누수
public class MemoryLeak {
    private static List<Object> cache = new ArrayList<>();
    
    public void processData(Object data) {
        cache.add(data); // 영구적으로 쌓임
    }
}
```

**원인:**
- 생성된 객체가 GC 대상이 되지 않음 (강한 참조 유지)
- 시간이 지날수록 메모리 사용량 증가
- 로컬과 다른 이유: **데이터량 차이** (개발DB vs 운영DB)

---

### Stack이 터지는 경우 (StackOverflowError)
```java
// ❌ 무한 재귀
public int fibonacci(int n) {
    return fibonacci(n - 1) + fibonacci(n - 2); // 스택 깊이 무한 증가
}

// ❌ 순환 참조 직렬화
@Data
public class Node {
    private Node parent;
    private Node child;
}
// parent <-> child 순환 참조 + 깊은 직렬화 시도
```

**원인:**
- 함수 호출이 반환되지 않고 계속 쌓임
- 각 호출마다 Stack frame 생성
- 로컬에서 안 터지는 이유: **테스트 케이스의 입력값이 작음**

## 핵심 개념 정리

### 📍 Heap 메모리
| 항목 | 특성 |
|------|------|
| **용도** | 모든 객체(Object) 인스턴스 저장 |
| **생명주기** | GC에 의해 자동 회수 |
| **공유** | 모든 스레드가 공유 |
| **크기** | `-Xmx` 옵션으로 제한 |
| **에러 시점** | 즉시 아님 (GC 후에도 부족할 때) |

### 📍 Stack 메모리
| 항목 | 특성 |
|------|------|
| **용도** | 메서드 호출, 지역변수, 참조값 |
| **생명주기** | 메서드 반환 시 자동 해제 |
| **독립성** | **각 스레드마다 독립적** |
| **크기** | `-Xss` 옵션으로 제한 (보통 1MB) |
| **에러 시점** | 호출 깊이가 깊어질수록 |

### 📍 Method Area (Class Area)
- 클래스 정의, 메서드 바이트코드, 상수풀 저장
- 문제 상황에서는 거의 관련 없음

### ⚠️ 로컬 vs 프로덕션 차이 원인

```
┌─────────────────────────────────────────────┐
│  로컬                                        │
├─────────────────────────────────────────────┤
│ ✓ 작은 테스트 데이터 (몇 개 행)              │
│ ✓ -Xmx 크기 (보통 2GB 이상)                │
│ ✓ 제한된 스레드 (몇 개)                     │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  프로덕션                                    │
├─────────────────────────────────────────────┤
│ ✗ 대량의 실제 데이터 (100만 행 이상)        │
│ ✗ 제한된 -Xmx (보통 1GB 이상 4GB 이하)     │
│ ✗ 많은 동시 요청 (1000+ 스레드)             │
│ ✗ 메모리 누수 (시간이 지날수록 악화)        │
└─────────────────────────────────────────────┘
```

## 최종 결론

| 항목 | Heap | Stack |
|------|------|-------|
| **원인** | 객체 생성 > GC 속도 | 호출 깊이 > Stack 크기 |
| **에러** | `OutOfMemoryError: Java Heap space` | `StackOverflowError` |
| **디버깅** | 힙 덤프, GC 로그 | 스택 트레이스 |
| **해결** | 메모리 누수 제거, 캐시 정책 | 재귀 제거, 구조 변경 |
