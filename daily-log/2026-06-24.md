# JVM 메모리 구조 완벽 가이드

## 모범 답안

### 1단계: 메모리 영역 직관적 이해

JVM 메모리를 **"회사 건물"**에 비유하면 이해하기 쉽습니다:

```
┌─────────────────────────────────────────────────────┐
│                    JVM 메모리                         │
├─────────────────────────────────────────────────────┤
│                                                       │
│  📚 Method Area (클래스 보관소)                      │
│  ├─ 클래스 구조 (필드, 메서드, 생성자)               │
│  ├─ 상수 풀 (Constant Pool)                          │
│  └─ Static 변수                                      │
│                                                       │
├─────────────────────────────────────────────────────┤
│                                                       │
│  📦 Heap (객체 저장소) - GC 대상                      │
│  ├─ Young Generation (새 객체들)                     │
│  │  ├─ Eden (객체 탄생지)                            │
│  │  ├─ Survivor 0                                    │
│  │  └─ Survivor 1                                    │
│  ├─ Old Generation (장수 객체들)                     │
│  └─ Permanent/Metaspace (클래스 메타데이터)          │
│                                                       │
├─────────────────────────────────────────────────────┤
│                                                       │
│  🔥 Stack (실행 정보 저장소) - Thread별 독립적       │
│  ├─ Stack Frame 1 (메서드 A 호출)                   │
│  │  ├─ 로컬 변수: x=10                              │
│  │  ├─ 로컬 변수: name="John"                       │
│  │  └─ 반환 주소                                     │
│  ├─ Stack Frame 2 (메서드 B 호출)                   │
│  │  ├─ 로컬 변수: y=20                              │
│  │  └─ 참조값: obj → Heap의 객체                    │
│  └─ Stack Frame 3 (메서드 C 호출)                   │
│                                                       │
└─────────────────────────────────────────────────────┘
```

### 2단계: 저장 위치 명확히 하기

```java
public class MemoryExample {
    static String staticVar = "static";  // ✅ Method Area
    
    public static void main(String[] args) {
        int localInt = 10;                    // ✅ Stack
        String localStr = "Hello";            // ✅ Stack (참조값)
                                              // ✅ Heap ("Hello" 문자열 객체)
        
        Person person = new Person();         // ✅ Stack (참조값)
                                              // ✅ Heap (Person 객체)
        person.setName("Alice");
    }
}

class Person {
    private String name;                  // ✅ Heap (Person 객체 내부)
    
    public void setName(String name) {    // ✅ Stack (메서드 정보)
        this.name = name;
    }
}
```

**메모리 배치도:**
```
Stack:
┌──────────────┐
│ localInt: 10 │ ← 원시값은 Stack에 직접 저장
├──────────────┤
│ localStr: ───┼──→ Heap: "Hello" (String 객체)
├──────────────┤
│ person: ─────┼──→ Heap: Person 객체
│              │    ├─ name: ─┐
│              │    └────────┼──→ Heap: "Alice"
└──────────────┘
```

### 3단계: OutOfMemoryError 발생 원인 분석

#### **상황 1: "OutOfMemoryError: Java heap space"**
```java
// ❌ Heap 부족 (객체가 계속 생성되고 GC 안 됨)
List<Object> list = new ArrayList<>();
while (true) {
    byte[] data = new byte[1024 * 1024];  // 1MB 객체
    list.add(data);  // 계속 참조 유지 → GC 불가능
    // Heap 메모리 사용량: 점점 증가 → OutOfMemoryError
}
```

**원인:** 
- Heap에 생성된 객체들이 계속 참조 유지
- GC가 해제하지 못함
- Heap 크기 초과

#### **상황 2: Stack에서 함수 호출 반복**
```java
// ✅ Stack은 함수 호출 종료 시 자동 정리됨
public void recursiveCall(int depth) {
    int localVar = depth;  // Stack에 저장
    
    if (depth > 0) {
        recursiveCall(depth - 1);  // 새로운 Stack Frame 생성
    }
    // 함수 종료 시 Stack Frame 자동 제거 ✨
}
```

**차이점:**
- Stack: 함수 호출 → 정보 저장 → 함수 종료 → 자동 정리 (LIFO)
- Heap: 객체 생성 → GC 대상이 될 때까지 유지

## 핵심 개념 정리

### 📚 **Method Area (클래스 영역)**
- **저장 대상**: 클래스 구조, 메서드 코드, 상수, Static 변수
- **특성**: 
  - 프로그램 시작 시 로드, 종료 시까지 유지
  - 모든 Thread가 공유
  - 크기: 고정 (Java 8+ Metaspace는 Native Memory 사용)
- **문제**: 
  ```java
  // ⚠️ 동적 클래스 로드 반복 시 문제 발생
  ClassLoader classLoader = new URLClassLoader(...);
  for (int i = 0; i < 1000000; i++) {
      Class<?> clazz = classLoader.loadClass("DynamicClass");
  }
  // OutOfMemoryError: Metaspace (또는 PermGen)
  ```

### 📦 **Heap (객체 영역)**
- **저장 대상**: 모든 객체와 배열
- **특성**:
  - 동적 할당/해제
  - GC에 의해 자동 정리
  - Thread 간 공유 (동시성 주의)
  - 크기: JVM 옵션으로 조정 가능 (`-Xmx`, `-Xms`)
  
- **Generation 구조**:
  ```
  Young Gen (Eden → Survivor → Old Gen)
  - 대부분의 객체는 Young에서 빠르게 GC됨 (Minor GC)
  - 오래 살아남은 객체만 Old로 이동 (Major GC)
  ```

- **메모리 누수 패턴**:
  ```java
  // ❌ Pattern 1: Collection에 계속 추가
  static List<Object> cache = new ArrayList<>();
  public void addCache(Object obj) {
      cache.add(obj);  // 계속 쌓임 → GC 불가
  }
  
  // ❌ Pattern 2: Strong Reference 유지
  class MyListener {
      private Owner owner;  // 순환 참조
  }
  owner.listener = new MyListener(owner);
  
  // ❌ Pattern 3: Thread Local 누수
  ThreadLocal<byte[]> threadLocal = new ThreadLocal<>();
  threadLocal.set(new byte[10 * 1024 * 1024]);
  // Thread 종료 시 threadLocal.remove() 안 하면 누수
  ```

### 🔥 **Stack (호출 정보 영역)**
- **저장 대상**: 
  - 메서드 호출 정보 (Stack Frame)
  - 원시값 (int, boolean, double 등)
  - 객체/배열 참조값 (주소)
  
- **특성**:
  - Thread별로 독립적인 Stack 생성
  - LIFO (Last In First Out) 구조
  - 자동 메모리 관리 (함수 종료 시 자동 정리)
  - 크기 제한: `-Xss` 옵션으로 조정
  
- **Stack Overflow 원인**:

  // ❌ 무한 재귀
  public int fibonacci(int n) {
      return fibonacci(n-1) + fibonacci(n-2);  
      // StackOverflowError (Stack Frame이 계속 쌓임)
  }
  
  // ❌ 깊은 호출 체인
  void
