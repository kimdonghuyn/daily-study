### 1단계: 문제의 본질 파악

```java
// ❌ AOP 없이 작성한 코드 - 반복과 산재
@PostMapping("/posts")
public ResponseEntity createPost(@RequestBody PostRequest req) {
    // 1. 인증 확인 (반복)
    if (!isLoggedIn()) {
        throw new UnauthorizedException();
    }
    
    // 2. 권한 확인 (반복)
    if (!hasPermission(getCurrentUser())) {
        throw new ForbiddenException();
    }
    
    // 3. 로깅 (반복)
    log.info("게시물 작성 시도: {}", getCurrentUser());
    
    // 4. 실제 비즈니스 로직 (5줄)
    Post post = postService.createPost(req);
    
    // 5. 감시/모니터링 (반복)
    recordMetric("post.created");
    return ResponseEntity.ok(post);
}

@PostMapping("/register")
public ResponseEntity register(@RequestBody RegisterRequest req) {
    // 위와 동일한 인증/로깅/모니터링 코드 반복...
    if (!isLoggedIn()) { ... }
    log.info("회원가입 시도: {}", ...);
    // 실제 로직은 3줄...
}
```

**문제점:**
- 🔴 **횡단 관심사 산재**: 인증, 로깅, 모니터링이 여러 메서드에 흩어짐
- 🔴 **비즈니스 로직 희석**: 핵심 로직이 부가 기능에 묻혀있음
- 🔴 **유지보수 악화**: 로깅 정책 변경 시 모든 메서드 수정 필요
- 🔴 **단일 책임 원칙 위반**: 한 메서드가 여러 책임 담당

### 2단계: Spring AOP의 해결책

```java
// ✅ AOP를 통한 깔끔한 분리
@Aspect
@Component
public class SecurityAspect {
    
    // 횡단 관심사: 인증 확인
    @Before("@annotation(com.example.RequireLogin)")
    public void checkLogin(JoinPoint jp) {
        if (!isLoggedIn()) {
            throw new UnauthorizedException();
        }
    }
    
    // 횡단 관심사: 로깅
    @Before("execution(* com.example.service.*.*(..)) && @annotation(com.example.Loggable)")
    public void logging(JoinPoint jp) {
        log.info("메서드 실행: {}, 사용자: {}", 
            jp.getSignature(), getCurrentUser());
    }
    
    // 횡단 관심사: 성능 모니터링
    @Around("@annotation(com.example.Monitor)")
    public Object monitor(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.currentTimeMillis();
        try {
            return pjp.proceed();
        } finally {
            long elapsed = System.currentTimeMillis() - start;
            recordMetric(pjp.getSignature().getName(), elapsed);
        }
    }
}

// ✅ 비즈니스 로직에만 집중
@PostMapping("/posts")
@RequireLogin        // 선언적으로 정책 적용
@Loggable
@Monitor
public ResponseEntity createPost(@RequestBody PostRequest req) {
    // 비즈니스 로직만 명확하게
    Post post = postService.createPost(req);
    return ResponseEntity.ok(post);
}

@PostMapping("/register")
@RequireLogin
@Loggable
public ResponseEntity register(@RequestBody RegisterRequest req) {
    User user = userService.register(req);
    return ResponseEntity.ok(user);
}
```

**개선사항:**
- ✅ **관심사 분리**: 인증/로깅/모니터링이 별도의 클래스로 분리
- ✅ **비즈니스 로직 명확화**: 핵심 로직만 남음
- ✅ **재사용성**: 한 번 정의한 정책을 여러 메서드에 적용
- ✅ **유지보수성**: 정책 변경 시 한 곳에서만 수정

## 핵심 개념 정리

### AOP의 용어와 개념

| 용어 | 의미 | 예시 |
|------|------|------|
| **Concern** | 처리해야 할 문제 | 인증, 로깅, 트랜잭션 |
| **Cross-Cutting Concern** | 여러 곳에 산재된 관심사 | 모든 메서드에 필요한 인증 |
| **Aspect** | 공통 관심사를 처리하는 모듈 | `@Aspect` 클래스 |
| **Pointcut** | AOP를 적용할 지점 선택 | `@annotation(...)`, `execution(...)` |
| **Advice** | 실제 수행할 로직 | `@Before`, `@After`, `@Around` |
| **Join Point** | 실제 적용되는 지점 | 메서드 호출 시점 |

### Pointcut 표현식 종류

```java
// 1. 애노테이션 기반 (권장: 명시적이고 읽기 쉬움)
@Before("@annotation(com.example.RequireLogin)")

// 2. 실행 시점 기반
@Before("execution(* com.example.service.UserService.register(..))")

// 3. 조합
@Before("execution(public * *(..)) && @annotation(Loggable)")
```

### Advice 타입별 특징

```java
@Aspect
public class AdviceExample {
    
    // 메서드 실행 전 실행 (반환값 접근 불가)
    @Before("pointcut()")
    public void before(JoinPoint jp) { }
    
    // 메서드 정상 완료 후 실행
    @AfterReturning(pointcut = "pointcut()", returning = "result")
    public void afterReturning(Object result) { }
    
    // 메서드 예외 발생 시 실행
    @AfterThrowing(pointcut = "pointcut()", throwing = "ex")
    public void afterThrowing(Exception ex) { }
    
    // 메서드 완료 후 항상 실행 (finally처럼)
    @After("pointcut()")
    public void after() { }
    
    // 메서드 실행 전후를 모두 제어 (가장 강력)
    @Around("pointcut()")
    public Object around(ProceedingJoinPoint pjp) throws Throwable {
        // 전
        Object result = pjp.proceed(); // 메서드 실행
        // 후
        return result;
    }
}
```

// 📌 사례 1: 인증 확인
@Aspect
@Component
public class AuthAspect {
    
    @Before("@annotation(RequireLogin)")
    public void validateLogin() {
        Authentication auth = SecurityContextHolder.getContext()
            .getAuthentication();
        if (auth == null || !auth.isAuthenticated()) {
            throw new UnauthorizedException("로그인 필요");
        }
    }
}

// 📌 사례 2: 트랜잭션 관리
@Aspect
@Component
public class TransactionAspect {
    
    @Around("@annotation(Transactional)")
    public Object manageTransaction(ProceedingJoinPoint pjp) 
            throws Throwable {
        Transaction tx = transactionManager.begin();
        try {
            Object result = pjp.proceed();
            tx.commit();
            return result;
        } catch (Exception e) {
            tx.rollback();
            throw e;
        }
    }
}

// 📌 사례 3: 성능 모니터링
@Aspect
@Component
public class PerformanceAspect {
    
    @Around("execution(* com.example.service..*(..))")
    public Object measureTime(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.nanoTime();
        try {
            return pjp.proceed();
        } finally {
            long elapsed = System.nanoTime() - start;
            if (elapsed > 1_000_000_000) { // 1초 이상
                log.warn("느린 쿼리 감지: {} - {}ms",
                    pjp.getSignature(), elapsed / 1_000_000);
            }
        }
    }
}
