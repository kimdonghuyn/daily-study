# 🎯 JVM 메모리 구조 학습 정리

## ✅ 핵심 개념 요약
JVM 메모리는 **Stack(스택)**, **Heap(힙)**, **Method Area(메서드 영역)** 3가지로 구성된다. Stack은 지역변수와 메서드 호출을 관리하고, Heap은 객체 인스턴스를 동적으로 할당하며, Method Area는 클래스 정보와 상수를 저장한다.

## 🔑 놓치지 말아야 할 포인트
- **Stack vs Heap**: Stack은 LIFO로 자동 해제되고, Heap은 GC가 필요한 영역
- **Method Area의 역할**: 모든 스레드가 공유하는 클래스 메타데이터, 상수풀, 정적변수 저장소
- **메모리 누수 위험**: Heap에 할당된 객체가 참조 해제되지 않으면 GC 대상이 되지 않음

## 📚 내일 복습 포인트
1. Garbage Collection(GC) 동작 원리와 종류 (Serial, Parallel, CMS 등)
2. OutOfMemoryError 발생 상황별 분석 (HeapSpace, StackOverflow)
3. 메모리 프로파일링 도구
