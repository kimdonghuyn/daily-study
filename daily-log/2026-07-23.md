### 1. 🔧 문제 상황

# 상속과 다형성: 게임 개발 시나리오 해답

## 모범 답안

상속과 다형성을 활용하면 **기존 코드 수정 없이** 새로운 캐릭터 타입을 자유롭게 추가할 수 있습니다.

### 핵심 원리

```java
// 1. 부모 클래스: 모든 게임 객체의 공통 인터페이스 정의
public abstract class GameObject {
    protected float x, y;
    protected float health;
    
    // 구현을 강제하는 추상 메서드
    public abstract void move();
    public abstract void takeDamage(float damage);
    public abstract void draw();
    
    // 공통 로직
    public boolean isAlive() {
        return health > 0;
    }
}

// 2. 자식 클래스들: 각각의 구체적인 동작 구현
public class Player extends GameObject {
    @Override
    public void move() {
        // 사용자 입력 기반 이동
        handleUserInput();
    }
    
    @Override
    public void takeDamage(float damage) {
        health -= damage;
        showDamageEffect(); // 플레이어만의 특수 효과
    }
}

public class EnemyNPC extends GameObject {
    private AIBehavior aiBehavior;
    
    @Override
    public void move() {
        // AI 알고리즘으로 자동 이동
        aiBehavior.calculateNextPosition(x, y);
    }
    
    @Override
    public void takeDamage(float damage) {
        health -= damage;
        if (health <= 0) dropLoot(); // 적만의 드롭 로직
    }
}

public class Boss extends GameObject {
    private BossPhase currentPhase;
    
    @Override
    public void move() {
        // 페이즈별 특수 패턴
        currentPhase.executePattern();
    }
    
    @Override
    public void takeDamage(float damage) {
        // 보스만의 데미지 감소 메커니즘
        health -= damage * currentPhase.getDefenseMultiplier();
        if (health <= 0) triggerBossDeathEvent();
    }
}

// 3. 다형성의 마법: 타입에 상관없이 통일된 방식으로 처리
public class GameEngine {
    private List<GameObject> gameObjects = new ArrayList<>();
    
    public void updateGame() {
        // 모든 객체를 일괄 처리 - 실제 타입에 맞는 메서드가 자동 호출됨
        for (GameObject obj : gameObjects) {
            obj.move();      // Player는 playerMove, Enemy는 aiMove, Boss는 patternMove 실행
            obj.draw();      // 각각의 draw() 구현이 호출됨
        }
    }
    
    public void handleDamage(GameObject target, float damage) {
        // 대상이 누구든 관계없이 동일 코드
        target.takeDamage(damage);  // 알아서 올바른 takeDamage() 실행
    }
    
    // 새로운 캐릭터 타입 추가 시 이 코드는 수정 불필요!
    public void addGameObject(GameObject obj) {
        gameObjects.add(obj);
    }
}
```

### 이점 정리

| 관점 | 이점 |
|------|------|
| **기존 코드 변경** | ❌ 전혀 수정 불필요. GameEngine 로직은 그대로 유지 |
| **새 타입 추가** | ✅ GameObject 상속받아 메서드만 구현하면 끝 |
| **코드 중복** | ✅ 공통 로직(`isAlive()`)은 부모에 한 번만 작성 |
| **타입 검사 제거** | ✅ `if(obj instanceof Player)` 같은 타입 체크 불필요 |
| **유지보수성** | ✅ 각 클래스는 자신의 책임만 집중 (SRP) |

## 핵심 개념 정리

### 📌 상속 (Inheritance)
- **목적**: 공통된 속성과 메서드를 부모 클래스에 정의하여 중복 제거
- **예시**: 모든 게임 객체가 갖는 `x, y, health`, `isAlive()`
- **주의**: "IS-A 관계"가 명확해야 함 (Player IS-A GameObject)

### 📌 다형성 (Polymorphism)
- **개념**: 같은 메시지(메서드 호출)에 대해 객체마다 다르게 응답
- **구현 방식**:
  - **오버라이딩**: 부모 메서드를 자식이 재정의
  - **오버로딩**: 같은 이름, 다른 매개변수 (게임 예제에서는 오버라이딩이 핵심)

### 📌 추상 클래스 vs 인터페이스
```java
// 추상 클래스: "IS-A" 관계, 상태(필드) 공유 가능
public abstract class GameObject {
    protected float health;  // ← 공통 상태
    public abstract void move();
}

// 인터페이스: 계약(규칙)만 정의, 다중 구현 가능
public interface Drawable {
    void draw();
}

// 실무: 보통 추상 클래스로 기본 구조 잡고, 인터페이스로 추가 기능 명시
public class Player extends GameObject implements Drawable, Saveable {
    // ...
}
```

### 🎮 게임 엔진 구조 예제 (확장 가능한 설계)

```java
// ✅ 실무 수준: 이벤트 시스템과 결합
public abstract class GameObject implements EventPublisher {
    protected List<GameEventListener> listeners = new ArrayList<>();
    
    public void takeDamage(float damage) {
        float actualDamage = calculateDamage(damage);
        health -= actualDamage;
        
        // 모든 타입에서 일관된 이벤트 발행
        publishEvent(new DamageEvent(this, actualDamage));
        
        if (health <= 0) {
            publishEvent(new DeathEvent(this));
            onDeath();  // 타입별 특수 처리
        }
    }
    
    protected abstract void onDeath();  // 플레이어는 게임오버, 적은 경험치 드롭
    protected abstract float calculateDamage(float baseDamage);  // 방어력 계산 방식 다름
}

// ✅ 새로운 타입 추가 (15분이면 충분)
public class Minion extends GameObject {
    @Override
    protected void onDeath() {
        publishEvent(new MinionDeathEvent());
    }
    
    @Override
    protected float calculateDamage(float baseDamage) {
        return baseDamage; // 특수한 방어 메커니즘 없음
    }
}
```
