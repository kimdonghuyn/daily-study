## 📚 모범 답안 & 개념 정리

### 1. 📖 개념 설명

# 상속과 다형성을 통한 게임 캐릭터 설계

## 모범 답안

이 문제는 **상속**과 **다형성**의 가장 전형적인 사용 사례입니다.

### 1단계: 상속을 통한 공통 인터페이스 정의

```python
# 부모 클래스: 모든 게임 객체의 공통 구조 정의
class GameCharacter:
    def __init__(self, name, hp):
        self.name = name
        self.hp = hp
    
    # 추상 메서드: 자식 클래스가 반드시 구현해야 함
    def attack(self):
        raise NotImplementedError("attack() 메서드를 구현해야 합니다")
    
    def take_damage(self, damage):
        self.hp -= damage
        print(f"{self.name}이(가) {damage}의 피해를 입었습니다. (남은 HP: {self.hp})")
```

### 2단계: 다형성을 통한 구체적 구현

```python
# 캐릭터: 검으로 공격
class Player(GameCharacter):
    def attack(self):
        damage = 15
        print(f"⚔️  {self.name}이(가) 검으로 공격합니다! (데미지: {damage})")
        return damage

# 몬스터: 발톱으로 공격
class Monster(GameCharacter):
    def attack(self):
        damage = 8
        print(f"🐾 {self.name}이(가) 발톱으로 공격합니다! (데미지: {damage})")
        return damage

# 보스: 마법으로 공격
class Boss(GameCharacter):
    def attack(self):
        damage = 25
        print(f"✨ {self.name}이(가) 마법으로 공격합니다! (데미지: {damage})")
        return damage
```

### 3단계: 다형성의 진정한 가치 - 통일된 인터페이스로 처리

```python
# 핵심! 어떤 객체든 동일하게 처리 가능
class BattleSystem:
    @staticmethod
    def execute_turn(attackers):
        """모든 캐릭터 타입을 동일한 로직으로 처리"""
        for attacker in attackers:
            # 각 객체가 자신의 attack() 메서드를 실행
            # → 실제 타입에 맞는 구현이 자동 호출됨 (다형성)
            damage = attacker.attack()
    
    @staticmethod
    def simulate_battle(player, enemies):
        """게임 루프: 타입을 신경 쓰지 않음"""
        all_characters = [player] + enemies
        
        round_num = 1
        while player.hp > 0 and any(e.hp > 0 for e in enemies):
            print(f"\n=== 라운드 {round_num} ===")
            
            # 모든 캐릭터가 공격 → 다형성!
            for character in all_characters:
                if character.hp > 0:
                    damage = character.attack()
                    # 다른 캐릭터에게 데미지 적용
                    target = enemies[0] if character == player else player
                    target.take_damage(damage)
            
            round_num += 1

# 사용 예시
player = Player("용사", hp=100)
monsters = [
    Monster("슬라임", hp=20),
    Monster("오크", hp=30),
    Boss("드래곤", hp=150)
]

BattleSystem.simulate_battle(player, monsters)
```

### 실행 결과
```

⚔️  용사이(가) 검으로 공격합니다! (데미지: 15)
🐾 슬라임이(가) 발톱으로 공격합니다! (데미지: 8)
🐾 오크가(가) 발톱으로 공격합니다! (데미지: 8)
✨ 드래곤이(가) 마법으로 공격합니다! (데미지: 25)
```

## 핵심 개념 정리

### 📌 상속 (Inheritance)
- **목적**: 공통된 속성과 메서드를 부모 클래스에 정의
- **장점**: 코드 중복 제거, 계층적 구조 표현
- **예시**: `GameCharacter` ← `Player`, `Monster`, `Boss`

### 📌 다형성 (Polymorphism)
- **정의**: 같은 메서드 이름으로 다른 행동을 수행
- **종류**:
  - **메서드 오버라이딩** (Override): 자식이 부모 메서드를 재정의
  - **메서드 오버로딩** (Overload): 같은 이름의 메서드 중복 정의 (Python에선 제한적)

### 📌 DRY 원칙 (Don't Repeat Yourself)
```python
# ❌ 다형성 없이: 타입별로 반복되는 코드
if isinstance(character, Player):
    player_attack()
elif isinstance(character, Monster):
    monster_attack()
elif isinstance(character, Boss):
    boss_attack()

# ✅ 다형성으로: 통일된 코드
character.attack()  # 자동으로 올바른 구현 호출
```

### 📌 리스코프 치환 원칙 (Liskov Substitution Principle)
- 자식 클래스는 부모 클래스를 완전히 대체할 수 있어야 함
- `BattleSystem`에서 `GameCharacter` 타입이면 어떤 자식도 수용 가능

## 실무 포인트

### 1️⃣ **확장성 (Extensibility)**
```python
# 새로운 캐릭터 타입 추가 시 기존 코드 수정 불필요
class Archer(GameCharacter):
    def attack(self):
        damage = 12
        print(f"🏹 {self.name}이(가) 화살을 쏩니다!")
        return damage

# BattleSystem은 그대로 사용 가능!
# → Open/Closed Principle (확장에 열려있고 수정에 닫혀있음)
```

### 2️⃣ **타입 안정성 검증**
```python
from abc import ABC, abstractmethod

# Python 3.9+: ABC로 강제성 확보
class GameCharacter(ABC):
    @abstractmethod
    def attack(self):
        """반드시 구현해야 하는 메서드"""
        pass

# class BuggyClass(GameCharacter):
#     pass  # ❌ TypeError: Can't instantiate abstract class
```

### 3️⃣ **실무 사례**

| 시나리오 | 활용 |

| **API 응답 처리** | Request/Response 기본 클래스 → 각 API별 구현 |
| **데이터베이스 연결** | Database 기본 클래스 → MySQL, PostgreSQL, MongoDB 구현 |
| **결제 시스템** | PaymentGateway → Stripe, PayPal, KakaoPay 구현 |
| **알림 시스템** | Notifier → Email, SMS, Push 구현 |
