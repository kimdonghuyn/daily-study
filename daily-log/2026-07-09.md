### 1. 🔧 문제 상황

# 상속과 다형성 - 게임 캐릭터 시스템 설계

## 모범 답안

이 문제는 **상속(Inheritance)과 다형성(Polymorphism)**으로 해결합니다.

### 1단계: 기본 구조 설계

```java
// ✅ 공통 부분을 추상 클래스로 정의
public abstract class Character {
    protected String name;
    protected int hp;
    protected float speed = 1.0f;
    protected int attackPower;
    
    // 공통 메서드 - 모든 캐릭터가 공유
    public void move() {
        System.out.println(name + " moves at speed: " + speed);
    }
    
    // 추상 메서드 - 각 캐릭터마다 다르게 구현
    public abstract void attack();
    public abstract void die();
    
    // 공통으로 사용되는 메서드
    public void takeDamage(int damage) {
        hp -= damage;
        System.out.println(name + " takes " + damage + " damage. HP: " + hp);
    }
}
```

### 2단계: 구체적인 캐릭터 구현

```java
// 플레이어
public class Player extends Character {
    public Player(String name) {
        this.name = name;
        this.hp = 100;
        this.attackPower = 15;
        this.speed = 1.0f;
    }
    
    @Override
    public void attack() {
        System.out.println(name + " swings sword with power: " + attackPower);
    }
    
    @Override
    public void die() {
        System.out.println(name + " is defeated! Game Over!");
    }
}

// NPC
public class NPC extends Character {
    public NPC(String name) {
        this.name = name;
        this.hp = 50;
        this.attackPower = 5;
        this.speed = 0.5f;
    }
    
    @Override
    public void attack() {
        System.out.println(name + " swings staff gently with power: " + attackPower);
    }
    
    @Override
    public void die() {
        System.out.println(name + " faints...");
    }
}

// 보스
public class Boss extends Character {
    private int phase = 1;
    
    public Boss(String name) {
        this.name = name;
        this.hp = 500;
        this.attackPower = 50;
        this.speed = 0.8f;
    }
    
    @Override
    public void attack() {
        System.out.println(name + " unleashes devastating attack! Power: " + (attackPower * phase));
    }
    
    @Override
    public void die() {
        System.out.println(name + " crumbles to dust... Phase " + phase + " complete!");
    }
}
```

### 3단계: 다형성을 통한 통일된 관리

```java
public class GameEngine {
    private List<Character> characters = new ArrayList<>();
    
    public void addCharacter(Character character) {
        characters.add(character);
    }
    
    // ✅ 핵심: 모든 캐릭터를 Character 타입으로 관리
    public void updateAllCharacters() {
        for (Character character : characters) {
            character.move();      // 다형성: 각 캐릭터의 고유 방식으로 실행
            character.attack();
        }
    }
    
    // ✅ 한 곳에서만 수정하면 모든 캐릭터에 반영됨
    public void speedUpAllCharacters(float multiplier) {
        for (Character character : characters) {
            character.speed *= multiplier;
        }
    }
}
```

### 4단계: 사용 예시

```java
public class Main {
    public static void main(String[] args) {
        GameEngine engine = new GameEngine();
        
        // 다양한 캐릭터 추가
        engine.addCharacter(new Player("Hero"));
        engine.addCharacter(new NPC("Merchant"));
        engine.addCharacter(new Boss("Dragon"));
        
        System.out.println("=== Initial State ===");
        engine.updateAllCharacters();
        
        System.out.println("\n=== Speed Up 2x ===");
        engine.speedUpAllCharacters(2.0f);
        engine.updateAllCharacters();
    }
}
```

## 핵심 개념 정리

### 🏗️ **상속 (Inheritance)**
- **목적**: 공통된 속성과 기능을 부모 클래스에 정의
- **이점**: 코드 중복 제거, 유지보수 용이
- **구조**: `class Child extends Parent`
- **범위**: 단일 상속만 가능 (다중상속 불가)

### 🎭 **다형성 (Polymorphism)**
- **정의**: 같은 인터페이스로 서로 다른 동작을 구현
- **종류**:
  - **메서드 오버라이딩**: 부모의 메서드를 자식이 재정의
  - **메서드 오버로딩**: 같은 이름, 다른 매개변수
- **실무 가치**: 객체 타입에 관계없이 동일하게 처리 가능

### 📋 **추상 클래스 vs 인터페이스**
| 구분 | 추상 클래스 | 인터페이스 |
|------|----------|---------|
| **목적** | 공통 기능 + 상태 | 계약/명세 정의 |
| **구현** | 메서드 구현 가능 | Java8+ 기본 메서드 가능 |
| **상속** | 단일 상속 | 다중 구현 가능 |
| **사용 시기** | "~이다" (IS-A) | "~할 수 있다" (CAN-DO) |

### 🔄 **SOLID 원칙 관련**
- **Open/Closed Principle**: 확장에 열려있고 수정에 닫혀있음
- **Liskov Substitution**: 자식은 부모를 완벽히 대체 가능해야 함
- **Dependency Inversion**: 구체적 클래스 아닌 추상화에 의존
