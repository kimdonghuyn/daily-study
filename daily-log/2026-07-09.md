# 상속과 다형성 학습 정리

## ✅ 핵심 개념 요약
**상속**은 부모 클래스의 속성과 메서드를 자식 클래스가 물려받는 메커니즘이고, **다형성**은 같은 인터페이스로 다양한 객체를 다룰 수 있게 해주는 원칙입니다. 이를 통해 코드 재사용성을 높이고 유지보수를 용이하게 합니다.

## 🔑 놓치지 말아야 할 포인트
- **메서드 오버라이딩**: 자식 클래스에서 부모의 메서드를 재정의하여 다형적 동작 구현
- **상속 체계 설계**: 과도한 상속 깊이는 피하고, IS-A 관계를 명확히 정의할 것
- **추상 클래스/인터페이스**: 다형성을 효과적으로 구현하는 핵심 도구로, 계약(contract) 정의

## 📚 내일 복습 포인트
- 실제 백엔드 프로젝트에서 상속을 적용할 때 주의사항 (깊은 상속 계층 피하기)
- 인터페이스 vs 추상 클
