## 📝 학습 요약

✅ 핵심 개념 요약
JVM은 바이트코드를 OS에 독립적으로 실행하며, Heap/Stack/Method Area로 메모리를 관리한다. GC는 "대부분의 객체는 금방 죽는다"는 전제 하에 Young/Old Generation으로 나눠 동작하며, Full GC 시 Stop-The-World가 발생해 응답 지연의 주범이 된다.

🔑 놓치지 말아야 할 포인트
  • Minor GC는 Eden이 꽉 찰 때 발동 — Survivor에서 age가 임계값(15) 넘으면 Old Gen으로 Promotion
  • Full GC = Stop-The-World → 서비스 지연 직결, G1GC가 Java 9+ 기본값
  • 실무 OOM 패턴: static 캐시 대용량 객체, 커넥션 미닫음, 힙 사이즈 과소 설정

📚 내일 복습 포인트: G1GC Region 분할 방식, STW 최소화 전략
